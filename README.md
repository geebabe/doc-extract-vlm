# OCR API (PaddleOCR + vLLM)

A FastAPI-based document information extraction service. This project combines the powerful text-spotting capabilities of PaddleOCR (with Vietnamese language support) with the advanced visual reasoning of Qwen3-VL-2B (served via vLLM) to perform highly precise, schema-constrained structured data extraction.

## Features

- **Two-Stage Extraction Pipeline**: Utilizes local PaddleOCR for preliminary text detection and recognition, feeding the normalized bounding box coordinates into Qwen3-VL-2B for enhanced visual-language processing. The OCR outputs are converted to JSON representation for robust parsing.
- **Dynamic Pydantic Schemas**: Extracted data is guaranteed to adhere strictly to predefined JSON schemas via OpenAI-compatible structured outputs.
- **Route-Based Schema Selection**: Different endpoints map automatically to different document schemas.
- **Support for Both Files & URLs**: Submit document images either as a direct file upload or via a public URL.
- **Built-in Image Preprocessing**: Automatically downscales large images, upscales small images, and deskews ID cards (using `cv2.minAreaRect`).
- **Observability**: Prometheus metrics enabled at `/metrics` (request histograms for OCR and VLM processing times, and correction counts), along with request ID tracking via `X-Request-ID` headers.

## Supported Document Schemas

The API currently supports the following extraction schemas:
- **Invoices** (`/extract/invoice/*`): Extracts vendor details, line items, subtotals, tax, and totals.
- **ID Cards** (`/extract/id_card/*`): Extracts Vietnamese citizen ID card information (ID number, full name, date of birth, etc.).
- **General Documents** (`/extract/others/*`): Fallback for general document extraction (title, issuer, date, abstract, and key-value pairs).

## Setup & Installation

### Prerequisites
- Python 3.10+
- vLLM instance serving `Qwen/Qwen3-VL-2B-Instruct` (or similar VLM)

### Environment Variables
Create a `.env` file in the root directory. The application uses `pydantic-settings` to load these configurations:

```env
PROJECT_NAME="OCR API"
VLLM_BASE_URL="https://your-vllm-server-url/v1"
VLLM_MODEL_NAME="Qwen/Qwen3-VL-2B-Instruct"
VLLM_API_KEY="your-api-key"

OCR_LANG="vi"
OCR_VERSION="PP-OCRv5"
OCR_TEXT_REC_SCORE_THRESH="0.5"
```

### Option 1: Run Locally (Without Docker)

1. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
   *(Note: You may need system libraries like `libgl1` and `libglib2.0-0` for OpenCV.)*

2. **Start the Server:**
   ```bash
   uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
   ```

### Option 2: Run with Docker

1. **Build the Image:**
   ```bash
   docker build -t ocr-api-paddle-vlm .
   ```

2. **Run the Container:**
   ```bash
   docker run -p 8000:8000 --env-file .env ocr-api-paddle-vlm
   ```

## API Documentation

Once the server is running, interactive API documentation is automatically generated by FastAPI:
- **Swagger UI**: [http://localhost:8000/docs](http://localhost:8000/docs)
- **ReDoc**: [http://localhost:8000/redoc](http://localhost:8000/redoc)

### Key Endpoints

| Method | Endpoint                     | Description                                           |
|--------|------------------------------|-------------------------------------------------------|
| `GET`  | `/health`                    | Health check endpoint.                                |
| `GET`  | `/metrics`                   | Prometheus metrics endpoint.                          |
| `POST` | `/extract/invoice/file`      | Upload an invoice image for extraction.               |
| `POST` | `/extract/invoice/url`       | Submit an invoice URL for extraction.                 |
| `POST` | `/extract/id_card/file`      | Upload an ID card image for extraction.               |
| `POST` | `/extract/id_card/url`       | Submit an ID card URL for extraction.                 |
| `POST` | `/extract/others/file`       | Upload a general document image for extraction.       |
| `POST` | `/extract/others/url`        | Submit a general document URL for extraction.         |

### Example Usage (cURL)

**Extracting from a URL:**
```bash
curl -X 'POST' \
  'http://localhost:8000/extract/invoice/url' \
  -H 'Content-Type: application/json' \
  -d '{
  "url": "https://example.com/sample-invoice.jpg"
}'
```

**Extracting from an uploaded file:**
```bash
curl -X 'POST' \
  'http://localhost:8000/extract/id_card/file' \
  -H 'accept: application/json' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@/path/to/your/id_card.jpg'
```
